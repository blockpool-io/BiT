Introduction

BiT is a ReSTful JSON API that allows anyone to quickly and easily integrate their existing systems with the blockchain.

Within these pages you can find documentation, simple code snippets and working examples to speed your integration and also help you understand the scope and utility of BiT.

  • How to use

How to use this guide

There are three ways you can use BiT, listed below. Within each section is a list of the functional "plugins" that are available. Click through to the area you are interested in. The working samples integrate with the Blockpool test system.

BiT Trusted

BiT Trusted is ideal if you already have authentication and want to integrate BiT seamlessly with your existing system.

In this case your user provides their permission for BiT to trust your requests for resources that you have permission for. Once trust has been established between your user and us you will be able to act without asking again. Of course the user can revoke this trust and we encourage you to offer that option via your own interface though the user will always be able to use our own myBlockpool interface to do this.

View Documentation

PaaS

PaaS is a very cut down implementation ideal if you do not want to give BiT any user information.

In this case you do not provide any identifiable information for your customer to BiT at all; all authentication is carried out by your own application. It allows you to develop any technology you want on top of the blockchain, leaving you free to develop your IP without distraction.
